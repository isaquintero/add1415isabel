Introducción

Las máquinas virtuales que vamos a utilizar son:

  • Windows 7.
  • Windows 2008 Server.
  • GNU/Linux Lubuntu.

    • Debemos poner las MV en "modo puente". Las máquinas que actuen de servidor o aquellas a las que nos vamos a conectar de forma repetida deben tener IP estática. Las IP's que podemos usar son (si somos el alumno del puesto XX):

  • 172.16.108.XX
  • 172.16.108.1XX: Usar esta IP para los servidores Windows
  • 172.16.108.2XX: Usar esta IP para los servidores GNU/Linux

    • Escritorio remoto con VNC

    Accedemos a la MV de Windows 7 e instalamos y configuramos VNC para poder acceder a una máquina remota.

    Luego hacemos la instalación en Windows 2008 Server.

    Y, finalmente lo instalamos en GNU/Linux Lubuntu para hacer todos los pasos requeridos en la práctica.

    Una vez instalado, debemos probar las conexiones entre:

  • Windows 7 - Windows 2008 Server.


  • Acceder a Windows desde Linux.


  • Acceder a Linux desde Linux.


  • Acceder a Linux desde Windows.


    • Escritorio Remoto con RDP.

    En este caso, no hay que instalar nada, puesto que las versiones actuales de Windows ya vienen con el software de escritorio remoto preinstalado. Debemos configurar escritorio remoto para poder acceder a otra máquina (Panel de control -> Sistema -> Configuración de Acceso Remoto).

    Probamos las conexiones remotas:

  • Acceder a Windows Server 2008 - desde Windows7:


  • Accedemos a Windows Server 2008 - desde Linux: en este caso debemos instalar el paquete remina a través del comando apt-get install remina.


  • Acceder a Linux - desde Windows: debemos instalar xrdp a través del comando apt-get install xrdp. En este caso, la conexión no funciona.

    • Servidor de Terminales.

    Necesitamos una MV con Windows Server que NO sea controlador de dominio.En W2k8server RC2, Terminal Service tiene un nombre diferente. Para instalarlo, entro en funciones y agrego la función TS.

    Creamos dos usuarios locales y los unimos al grupos de acceso remoto.

    Finalmente, probamos la conexión remota de escritorio RDP, desde el cliente Windows hacia el servidor Windows Server.

    Aplicaciones remotas mediante RemoteApp.

    • En las configuraciones usar las IP's de las máquinas y no su nombre NETBIOS.
    • No configurar Puerta de Enlace TS.
    • No usar certificados.
